(내용은 [rules.md](file:///g:/%EB%82%B4%20%EB%93%9C%EB%9D%BC%EC%9D%B4%EB%B8%8C/%EC%99%80%EC%9D%B4%EC%95%A4%EC%95%84%EC%B2%98/%EC%99%80%EC%9D%B4%EC%95%A4%EC%95%84%EC%B2%98_%EA%B5%90%EC%9C%A1/260608_%EB%B0%94%EC%9D%B4%EB%B8%8C%EC%BD%94%EB%94%A9%20%EA%B5%90%EC%9C%A1/rules.md)를 참고하십시오.)
2. `index.html` 파일을 생성하고 기본 구조를 만든 후 개발을 시작합니다.

## 서비스 1. 취미 빙고 생성기

취미생활을 더 오래, 더 다양하게 즐기도록 미션 빙고와 기록 회고를 제공하는 웹서비스입니다.

### 1. 유저

- **취미를 꾸준히 이어가고 싶은 개인 유저**: 영화, 독서, 전시, 음악 감상처럼 혼자 즐기는 취미가 있지만 금방 루틴이 반복되는 사람
- **취미 기록을 남기는 회고형 유저**: 노션, 다이어리, SNS 등에 감상평과 활동 기록을 남기며 자신의 취향 변화를 보고 싶은 사람
- **소모임 및 커뮤니티 운영자**: 영화 모임, 독서 모임, 챌린지 모임에서 멤버들이 함께 수행할 가벼운 미션이 필요한 운영자

### 2. 현상

- **취미가 반복 소비로 굳어짐**: 영화를 봐도 익숙한 장르만 보고, 책을 읽어도 비슷한 저자와 주제만 선택해 새로운 발견이 줄어듭니다.
- **기록이 흩어져 성장감이 약함**: 감상평, 완료 날짜, 미션 수행 여부가 각기 다른 앱이나 메모장에 남아 취미의 누적 경험이 잘 보이지 않습니다.
- **챌린지 설계가 번거로움**: "흑백 영화 보기", "3시간 넘는 영화 보기", "아카데미 작품상 보기"처럼 재미있는 미션을 매번 직접 떠올리기 어렵습니다.
- **완료 후 회고가 막막함**: 빙고를 끝내도 내가 어떤 취향을 새로 발견했는지, 어떤 활동을 편식했는지 정리하기 어렵습니다.

### 3. 방법론

- **매체**: 웹서비스 (순수 HTML/CSS/JS 바닐라 스택)
- **주요 기능 (1) - 취미별 미션 빙고 자동 생성**
  - 유저가 `영화`, `독서`, `음악`, `전시`, `운동` 등 취미 카테고리와 난이도를 선택하면 3x3 또는 5x5 빙고판을 생성합니다.
  - 영화 예시 미션: 흑백 영화 보기, 3시간 넘는 영화 보기, 아카데미 작품상 보기, 감독 데뷔작 보기, 같은 배우 출연작 2편 보기
  - 독서 예시 미션: 100쪽 이하 책 읽기, 번역서 읽기, 출간 10년 넘은 책 읽기, 처음 읽는 작가의 책 읽기, 한 문장 필사하기
- **주요 기능 (2) - 일자별/빙고별 기록 관리**
  - 각 칸을 클릭하면 수행 날짜, 감상 메모, 별점, 첨부 링크, 완료 상태를 저장할 수 있습니다.
  - 캘린더 뷰에서는 날짜별 활동을 보고, 빙고 뷰에서는 미션 달성률과 남은 칸을 한눈에 확인합니다.
- **주요 기능 (3) - 자동 회고 보고서**
  - 완료된 기록을 바탕으로 주간/월간/빙고별 회고를 자동 생성합니다.
  - 보고서에는 완료율, 가장 많이 수행한 유형, 새롭게 발견한 취향, 다음 빙고 추천 미션을 포함합니다.

### 4. 기대효과

- **취미 지속률 향상**: 막연히 "뭐 하지?"를 고민하는 시간을 줄이고, 다음 활동을 게임처럼 선택하게 만들어 취미 루틴을 유지합니다.
- **취향 확장**: 유저가 평소 선택하지 않던 장르, 시대, 작가, 형식으로 자연스럽게 이동해 취미 경험의 폭을 넓힙니다.
- **기록의 자산화**: 흩어진 감상과 수행 이력을 빙고 단위로 모아 나만의 취미 포트폴리오처럼 축적합니다.
- **커뮤니티 활용성 증가**: 같은 빙고판을 공유해 소모임 챌린지, SNS 인증, 월간 미션 이벤트로 확장할 수 있습니다.

## 서비스 2. 창의성 자판기

기존 아이디어 두 개 이상을 결합해 플롯, 숏폼, 서비스 아이디어의 초안을 빠르게 뽑아내는 웹서비스입니다.

### 1. 유저

- **웹툰/웹소설 작가 및 시나리오 작가**: 새로운 플롯, 세계관, 사건, 클리셰 비틀기가 필요한 창작자
- **숏폼 크리에이터**: 유튜브 쇼츠, 릴스, 틱톡에서 쓸 주제와 형식, 분위기를 계속 발굴해야 하는 사람
- **서비스 기획자 및 개발자**: 해커톤, 사이드 프로젝트, 신규 기능 기획에서 낯선 조합의 앱/웹 아이디어가 필요한 메이커
- **아이디어는 있지만 정리가 어려운 유저**: 떠오른 메모를 구조화된 기획안으로 다듬는 데 도움을 받고 싶은 사람

### 2. 현상

- **빈 화면 공포증**: 창작이나 기획을 시작할 때 첫 설정, 첫 문장, 첫 기능 정의를 잡지 못해 멈춥니다.
- **아이디어 자가 복제**: 익숙한 장르, 익숙한 문제, 익숙한 해결책 안에서만 생각해 결과물이 비슷해집니다.
- **이종 결합의 어려움**: "러닝 x 땅따먹기", "와인 x 평점"처럼 서로 다른 개념을 붙였을 때의 가능성을 빠르게 탐색하기 어렵습니다.
- **초안 완성까지의 간극**: 랜덤 키워드가 나와도 그것을 줄거리, 릴스 대본, 서비스 명세로 바꾸는 과정에서 다시 막힙니다.

### 3. 방법론

- **매체**: 웹서비스 (순수 HTML/CSS/JS 바닐라 스택)
- **주요 기능 (1) - AI 랜덤 키워드 자판기**
  - 유저가 자판기 유형을 선택하면 정해진 조합 공식에 따라 키워드를 추출합니다.
  - 플롯 자판기: `세계관/배경 x 인물 x 사건`
  - 플롯 자판기 2: `기존 작품 A x 기존 작품 B x 반전 요소`
  - 릴스 자판기: `주제 x 형식 x 분위기`
  - 솔루션 자판기: `관심사 x 문제/행동 x 서비스 형태`
- **주요 기능 (2) - 유저 아이디어 입력**
  - 추출된 키워드 아래에 유저가 떠오른 아이디어를 한 줄 또는 짧은 메모로 작성합니다.
  - 완성된 문장이 아니라 "러닝으로 동네를 점령하는 게임", "와인 초보용 평점 앱"처럼 거친 생각도 입력할 수 있게 합니다.
- **주요 기능 (3) - AI 완성 도움**
  - AI가 키워드와 유저 메모를 결합해 목적에 맞는 초안을 생성합니다.
  - 플롯은 로그라인과 3막 구조, 릴스는 훅/장면/자막/CTA, 솔루션은 타겟 유저/핵심 기능/MVP 화면 구성으로 정리합니다.
- **주요 기능 (4) - 저장 및 내보내기**
  - 마음에 드는 조합과 초안을 보관함에 저장하고, 텍스트 복사 또는 다운로드로 외부 문서에 옮길 수 있습니다.

### 4. 기대효과

- **창작 시작 장벽 감소**: 무작위 조합을 출발점으로 삼아 완벽한 아이디어가 없어도 바로 쓰기 시작할 수 있습니다.
- **신선한 조합 발굴**: 사람이 의도적으로 떠올리기 어려운 이종 결합을 통해 콘텐츠와 서비스 아이디어의 차별성을 높입니다.
- **기획 리드타임 단축**: 브레인스토밍부터 초안 아웃라인까지 걸리는 시간을 줄여 창작자와 메이커의 실행 속도를 높입니다.
- **AI 협업 경험 강화**: AI가 처음부터 대신 만드는 것이 아니라, 유저의 메모를 재료로 삼아 완성도를 높이는 공동 작업 흐름을 제공합니다.
