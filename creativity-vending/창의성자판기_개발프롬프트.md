## 역할 지시

당신은 **순수 바닐라 HTML/CSS/JS** 전문 개발 에이전트입니다.
아래 서비스 기획과 기술 규칙을 읽고, **창의성 자판기** 웹 앱을 처음부터 완성까지 구현해 주세요.

---

## 서비스 개요

**창의성 자판기**는 기존 아이디어 두 개 이상을 결합해  
플롯, 숏폼, 서비스 아이디어의 초안을 빠르게 뽑아내는 웹서비스입니다.

---

## 타겟 유저

| 유저 | 상황 |
|---|---|
| 웹툰/웹소설 작가·시나리오 작가 | 새로운 플롯·세계관·클리셰 비틀기가 필요한 창작자 |
| 숏폼 크리에이터 | 유튜브 쇼츠·릴스·틱톡 주제를 계속 발굴해야 하는 사람 |
| 서비스 기획자·개발자 | 해커톤·사이드 프로젝트에서 낯선 조합의 아이디어가 필요한 메이커 |
| 아이디어 정리가 어려운 유저 | 떠오른 메모를 구조화된 기획안으로 다듬고 싶은 사람 |

---

## 핵심 문제 (Pain Points)

- **빈 화면 공포증**: 첫 설정·첫 문장·첫 기능 정의를 잡지 못해 멈춥니다.
- **아이디어 자가 복제**: 익숙한 패턴 안에서만 생각해 결과물이 비슷해집니다.
- **이종 결합의 어려움**: "러닝 x 땅따먹기"처럼 다른 개념을 붙였을 때의 가능성을 탐색하기 어렵습니다.
- **초안 완성까지의 간극**: 랜덤 키워드가 나와도 그것을 실제 대본·명세로 바꾸는 과정에서 다시 막힙니다.

## 기능 명세

### 기능 1 — 자판기 유형 선택

화면 상단에서 유저가 자판기 유형을 선택합니다.

| 유형 키 | 이름 | 조합 공식 |
|---|---|---|
| `plot` | 플롯 자판기 | 세계관/배경 × 인물 유형 × 사건 |
| `plot2` | 플롯 자판기 2 | 기존 작품 A × 기존 작품 B × 반전 요소 |
| `reels` | 릴스 자판기 | 주제 × 형식 × 분위기 |
| `solution` | 솔루션 자판기 | 관심사 × 문제/행동 × 서비스 형태 |

### 기능 2 — 키워드 뽑기

- **[자판기 돌리기]** 버튼을 누르면 해당 유형의 슬롯에서 각 축마다 키워드 1개를 무작위 선택해 카드 형태로 표시합니다.
- 각 슬롯 키워드 풀은 JS 파일 내 배열로 관리합니다. (Supabase에 저장하지 않음)
- 키워드 풀 예시:
  - `plot` 세계관: "멸망 후 지하 도시", "SF 우주정거장", "조선 후기 한양", "가상 현실 게임 속", "기억을 파는 시장"
  - `plot` 인물: "기억을 잃은 탐정", "반항아 AI", "마지막 대장장이", "쌍둥이 스파이"
  - `plot` 사건: "금지된 기술이 발견됨", "주인공이 배신당함", "두 세계가 충돌함"
  - *(나머지 유형도 동등한 풍부함으로 직접 설계)*

### 기능 3 — 유저 메모 입력

- 키워드 카드 아래에 `<textarea>` 메모 입력창을 표시합니다.
- 플레이스홀더: `"러닝으로 동네를 점령하는 게임"처럼 거친 생각도 괜찮아요.`

### 기능 4 — 보관함 저장 (Supabase 연동)

- **[보관함에 저장]** 버튼을 누르면 현재 키워드 조합 + 유저 메모를 Supabase `creativity_vault` 테이블에 INSERT합니다.
- 저장 성공 시 토스트 알림 표시.
- 보관함 탭에서 저장된 카드 목록을 최신순으로 불러와 표시합니다. (SELECT)
- 각 카드에는 **[텍스트 복사]** 버튼을 제공합니다.

### 기능 5 — 텍스트 복사 / 다운로드

- 카드의 키워드 조합 + 메모를 클립보드에 복사하는 버튼을 제공합니다.

## 기술 규칙 (반드시 준수)

1. **순수 바닐라 스택**: 외부 JS 라이브러리(jQuery, React, Vue 등) 및 CSS 프레임워크(Tailwind, Bootstrap 등) 사용 금지. 표준 HTML5/CSS3/ES6+ 만 사용.
2. **ES Modules**: `<script type="module">`, 파일 간 `import` / `export` 사용.
3. **빌드 도구 없음**: npm, Webpack, Vite 사용 금지. 정적 파일만으로 구성.
4. **파일 500줄 상한**: 단일 파일이 400줄에 임박하면 기능별로 분리.
5. **폴더 구조** 준수:

```
📂 창의성-자판기/
├── index.html
├── css/
│   └── main.css
├── js/
│   ├── app.js          ← 전역 상태 · 초기화 · 탭 라우팅
│   ├── components/
│   │   ├── vendingCard.js    ← 키워드 카드 렌더링
│   │   └── vaultList.js      ← 보관함 목록 렌더링
│   └── utils/
│       ├── supabase.js       ← Supabase 클라이언트 · CRUD 함수
│       └── keywords.js       ← 자판기 유형별 키워드 풀 데이터
└── assets/
```

6. **상태 관리**: DOM 속성에 상태 저장 금지. `const state = {}` 단일 객체 사용.
7. **이벤트 위임**: 동적 요소는 부모에 이벤트 위임.
8. **비동기**: 모든 Supabase 호출은 `async/await` + `try/catch`.
9. **시맨틱 HTML**: `<header>`, `<main>`, `<section>`, `<footer>` 필수 사용.
10. **CSS 변수**: 색상·폰트·간격은 `:root` 변수로 관리.
